## Tutorials

| | |
|---|---|
| **Purpose** | A lesson where the learner acquires skill by doing |
| **Serves** | Acquisition of skill through action |
| **Analogy** | Teaching a child to cook |

**Must do:**

- Guide through concrete steps toward a meaningful result
- Show results early and often (within every 3 steps)
- Maintain a narrative of the expected: "You should see..."
- Be reliable and completable

**Must not do:**

- Explain why things work (link to Explanation)
- Offer choices or alternatives
- Include reference material for completeness

**Language:** "We will...", "First, do X. Now, do Y.", "You should see..."

## How-to Guides

| | |
|---|---|
| **Purpose** | Directions for solving a real-world problem |
| **Serves** | Application of skill through action |
| **Analogy** | A recipe in a cookbook |

**Must do:**

- Address a real-world task (defined by user need, not tool capability)
- Provide actionable steps
- Assume the reader knows what they want and has basic competence

**Must not do:**

- Teach (link to Tutorial)
- Explain why (link to Explanation)
- Include exhaustive reference material (link to Reference)

**Language:** Title starts with "How to...", "If you want X, do Y."

## Reference

| | |
|---|---|
| **Purpose** | Technical description of the machinery |
| **Serves** | Application of skill through cognition |
| **Analogy** | Information on the back of a food packet |

**Must do:**

- Describe neutrally and factually
- Mirror the structure of the thing being documented
- Use consistent formatting throughout
- Provide brief illustrative examples

**Must not do:**

- Instruct (link to How-to or Tutorial)
- Explain (link to Explanation)
- Contain opinions

**Language:** Declarative statements, factual lists, warning notices.

## Explanation

| | |
|---|---|
| **Purpose** | Discursive treatment that deepens understanding |
| **Serves** | Acquisition of skill through cognition |
| **Analogy** | An article on culinary social history |

**Must do:**

- Answer "why?" and "can you tell me about...?"
- Make connections to other concepts
- Provide context, background, design rationale
- Acknowledge multiple perspectives

**Must not do:**

- Provide step-by-step instructions (link to Tutorial or How-to)
- Include technical descriptions that belong in Reference

**Language:** "The reason for X is...", "W is better than Z because..."

## Cross-referencing

Link between quadrants. Do not embed.

| From | Link to | Reason |
|------|---------|--------|
| Tutorial | Explanation | "For more on why this works..." |
| Tutorial | Reference | "For the full list of options..." |
| How-to | Reference | "See the reference for all parameters." |
| How-to | Explanation | "For background on this approach..." |
| Reference | How-to | "For a guide on using this..." |
| Explanation | Tutorial | "To try this yourself..." |

## The Compass

Two questions classify any content:

1. Does this inform **action** or **cognition**?
2. Does this serve **acquisition** or **application**?

For background on why these quadrants exist, see
[The Diataxis Framework](/explanation/diataxis-framework/).
