Reference documentation describes the project's machinery: file
formats, command-line tools, and rules. Consult these when you need a
specific fact or specification.

- [Quadrant Rules](quadrant-rules/) — the rules each Diataxis quadrant
  must follow, with cross-referencing guidance and the compass questions.
- [diataxis.toml Schema](diataxis-toml-schema/) — every field in
  `diataxis.toml`, with type, requirement, and valid values.
- [Build Commands Reference](cli-reference/) — `make build`, `make serve`,
  `make exercises`, `make clean`, and bypassing `make` to run `hugo` directly.
- [Scoring Rubric](scoring-rubric/) — score scale, project- and file-level
  criteria, quadrant criteria, and the `scores.toml` output format.
